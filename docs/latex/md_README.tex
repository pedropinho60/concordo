Projeto de \char`\"{}\+Linguagem de Programação I\char`\"{}\hypertarget{md_README_autotoc_md1}{}\doxysection{Descrição}\label{md_README_autotoc_md1}
Esse projeto é uma \char`\"{}imitação\char`\"{} do \href{https://www.discord.com}{\texttt{ {\bfseries{Discord}}}}, feita em C++. Esse repositório contém as versões 1, 2 e 3 do projeto.

Na versão 1, é possível fazer login, criar, configurar e entrar em servidores. A versão 2 adiciona as funcionalidades de canais, permitindo criar canais de texto e voz dentro de um servidor, e enviar mensagens neles. A versão 3 adiciona funcionalidades de salvar e carregar, permitindo que o programa guarde informações mesmo após ser encerrado. Também é possível rodar o programa em dois terminais simultaneamente.\hypertarget{md_README_autotoc_md2}{}\doxysection{Autoria}\label{md_README_autotoc_md2}
O projeto foi feito por Pedro Vinícius Oliveira Maciel Pinho.\hypertarget{md_README_autotoc_md3}{}\doxysection{Comandos}\label{md_README_autotoc_md3}

A pasta \mbox{[}{\bfseries{data}}\mbox{]}(data) contém arquivos com informações de salvamento. Ao executar o programa, as informações são lidas do arquivo. Se quiser remover o conteúdo salvo, apague o conteúdo de ambos arquivos e substitua por uma única linha contendo {\ttfamily 0}.\hypertarget{md_README_autotoc_md10}{}\doxysection{Compilação e execução}\label{md_README_autotoc_md10}
Para compilar o programa, é utilizado o \href{https://cmake.org}{\texttt{ {\bfseries{CMake}}}}.

Na pasta principal do projeto, execute os comandos a seguir\+:


\begin{DoxyItemize}
\item {\ttfamily cmake -\/B build} para gerar os arquivos de compilação
\item {\ttfamily cmake -\/-\/build build} para compilar o programa
\end{DoxyItemize}

Após a compilação, o programa pode ser executado com {\ttfamily ./build/program}.\hypertarget{md_README_autotoc_md11}{}\doxysection{Limitações}\label{md_README_autotoc_md11}
O programa não verifica se o número de argumentos está correto para cada comando. Por exemplo, se um usuário digitar {\ttfamily create-\/user \texorpdfstring{$<$}{<}email\texorpdfstring{$>$}{>} \texorpdfstring{$<$}{<}senha\texorpdfstring{$>$}{>}}, e não especificar o nome do usuário, o programa criará um usuário com email e senha, mas com o nome vazio, ao invés de causar um erro.\hypertarget{md_README_autotoc_md12}{}\doxysection{Documentação}\label{md_README_autotoc_md12}
